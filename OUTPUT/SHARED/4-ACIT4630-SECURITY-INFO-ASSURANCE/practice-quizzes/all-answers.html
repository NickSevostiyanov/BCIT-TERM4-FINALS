Multi-SelectQuestion 1 of 30B, C, E
What is the main objective of information security? Select the THREE that best apply.
Explanation: CIA = Confidentiality + Integrity + Availability. These are the three main objectives of information security. Configuration and Identification are not part of the CIA triad.
MCQuestion 2 of 30B
Using redundant components (e.g. RAID technology) in a system enforces what goal of information security?
Explanation: RAID = Availability. Redundant components keep the system running even if one component fails. Common exam trap: students pick Integrity — RAID does NOT protect data integrity, it protects uptime.
MCQuestion 3 of 30C
Encryption protects __________, whereas hashing helps protect __________.
Explanation: Encryption → Confidentiality (only authorized parties can read data). Hashing → Integrity (detects if data was tampered with). These are frequently swapped on exams — know them cold.
MCQuestion 4 of 30A
Denial of service threat compromises __________, whereas tampering threat compromises __________ of a system.
Explanation: DoS → Availability (denies users access to services). Tampering → Integrity (modifies data). These are STRIDE threats mapped to CIA properties. Know your STRIDE threat-to-CIA mappings.
MCQuestion 5 of 30A
Which type of threat actor is normally the most sophisticated?
Explanation: APT (Advanced Persistent Threat) = most sophisticated actors, often nation-state backed with long-term persistent access goals. Script kiddies = least sophisticated (use existing tools, no custom development).
MCQuestion 6 of 30B
What does the "S" in STRIDE stand for?
Explanation: STRIDE = Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege. Used with data flow diagrams to identify threats.
MCQuestion 7 of 30C
What does CVE stand for?
Explanation: CVE = Common Vulnerabilities and Exposures (the identifier/catalog). CVSS = Common Vulnerability Scoring System (severity score). NVD = National Vulnerability Database (adds patch info and severity scores to CVEs).
MCQuestion 8 of 30C
Which stage of Vulnerability Management involves patching systems and fixing issues?
Explanation: Vulnerability Management lifecycle: Discover → Prioritize → Remediate → Validate → Monitor. Remediate = fix/patch. Validate = confirm fixes worked. Monitor = ongoing tracking.
MCQuestion 9 of 30C
KeyLogger, which monitors keystrokes typed by a user, is an example of what type of malicious software?
Explanation: Keylogger = Spyware (monitors/records user activity). Adware = unwanted ads. Trojan = disguised as legitimate software. Ransomware = encrypts files and demands payment.
MCQuestion 10 of 30B
Which one of the following types of malware can spread without any user interaction?
Explanation: Worm = spreads WITHOUT user interaction (self-propagating). Virus = requires user action (opening a file, running a program). Classic exam distinction — do not mix these up.
MCQuestion 11 of 30D
What type of phishing attack focuses specifically on senior executives of a targeted organization?
Explanation: Whaling = phishing targeting senior executives specifically. Spear phishing = targeted at a specific individual (not necessarily an exec). Pharming = DNS-based redirection. Vishing = voice/phone phishing.
MCQuestion 12 of 30B
Executing suspicious code in a contained environment before allowing it to run on the protected system is called __________.
Explanation: Sandboxing = running suspicious code in an isolated environment to observe behavior before allowing it to execute on the real system. Signature detection = matching known malware patterns.
MCQuestion 13 of 30C
What type of backup includes only those files that have changed since the most recent full OR incremental backup?
Explanation: Incremental = changes since last full OR incremental. Differential = changes since last FULL backup only. This is a tricky distinction — incremental chains from any previous backup, differential always goes back to the last full.
T/FQuestion 14 of 30A — True
Differential backup uses more storage than incremental backup, but is faster to recover since it only needs the full backup and the most recent differential backup.
Explanation: TRUE. Differential = more storage (grows with every change since last full), but faster restore (2 files: full + latest differential). Incremental = less storage (only changes since last backup), but slower restore (must merge entire chain of incrementals).
T/FQuestion 15 of 30B — False
All adverse security events are considered security incidents.
Explanation: FALSE. Not all adverse security events become security incidents. A security event = any observable occurrence (e.g., a login attempt, whether successful or not). A security incident = an adverse event that actually violates security policy. Events ⊃ Incidents.
MCQuestion 16 of 30C
How are public and private keys used in asymmetric cryptography?
Explanation: Asymmetric crypto: anything encrypted with one key → decrypted with the OTHER key from the same pair. This allows both secure messaging (encrypt with recipient's public key) and digital signatures (encrypt hash with sender's private key).
MCQuestion 17 of 30B
What is used to create a digital signature?
Explanation: Digital signature = HASH of the message encrypted with sender's PRIVATE key. Verifier decrypts signature with sender's PUBLIC key to get hash, then re-hashes message to confirm match. Proves authenticity + integrity.
Multi-SelectQuestion 18 of 30A, B, C
Select all that's true about a secure cryptographic hash function.
Explanation: Secure hash properties: fixed-length output (regardless of input), one-way (cannot reverse), collision resistant (hard to find two inputs with same hash). D and E are the OPPOSITE of correct — hashes are NOT reversible and collision resistance means it's hard, not easy, to find collisions.
MCQuestion 19 of 30C
What standard governs the structure and content of digital certificates?
Explanation: X.509 = the certificate standard (governs structure and content of digital certificates). X.500 = directory services standard. 802.1x = port-based network access control. 802.11ac = WiFi standard.
MCQuestion 20 of 30C
Which certificate type provides the lowest level of trust?
Explanation: Trust levels (lowest to highest): DV < OV < EV. DV only verifies domain ownership. OV verifies the organization. EV is the highest — verifies identity rigorously (was shown as green bar in browsers).
MCQuestion 21 of 30B
What is the most effective method of revoking digital certificates?
Explanation: OCSP = most effective revocation method (real-time check of certificate status). CRL = older method — a downloaded list of revoked certs that may be outdated. Certificate Pinning ≠ revocation (it prevents certs from changing).
MCQuestion 22 of 30C
What's the first step to initiate a TLS secure session between two systems?
Explanation: TLS handshake step 1 = CLIENT sends request with list of supported cipher suites. Then server selects one and sends back its certificate. Then key exchange happens. Then encrypted communication begins. Client initiates, not server.
MCQuestion 23 of 30C
Which one is NOT a secure protocol to transfer files between two systems?
Explanation: FTP = NOT secure (plaintext, port 21). Secure alternatives: SFTP (FTP over SSH, port 22), SCP (Secure Copy over SSH, port 22), FTPS (FTP over TLS, port 990). Never use plain FTP or Telnet.
MCQuestion 24 of 30C
Systems that must accept direct connections from the outside world (e.g., public web servers) should be placed on __________.
Explanation: DMZ (Demilitarized Zone) = network segment for public-facing servers (web, email, DNS). Isolated from the internal network so that if a DMZ server is compromised, attackers cannot directly reach internal systems.
MCQuestion 25 of 30C
What type of firewall rule error occurs when a service is decommissioned but the related firewall rules are not removed?
Explanation: Orphaned rule = leftover rule after a service is decommissioned (no longer needed but not cleaned up). Shadowed rule = rule that is unreachable because a previous rule already matches the traffic. Promiscuous rule = overly permissive rule.
T/FQuestion 26 of 30B — False
During the very first step of the TCP/IP handshake, a SYN/ACK packet is sent to the server.
Explanation: FALSE. The first step = SYN (client → server). Then server responds with SYN/ACK (step 2). Then client sends ACK (step 3). SYN/ACK is the SECOND step, not the first.
T/FQuestion 27 of 30B — False
Both TCP and UDP perform a three-way handshake before transmitting data.
Explanation: FALSE. Only TCP does the 3-way handshake (SYN → SYN/ACK → ACK). UDP is connectionless — no handshake at all. TCP = reliable (guaranteed delivery). UDP = unreliable but faster (used for voice/video/streaming).
MCQuestion 28 of 30C
An attacker inserted false DNS records to redirect you from your bank's website to a fake site. This is an example of:
Explanation: DNS Poisoning = injecting false DNS records to redirect users to malicious sites. URL Redirection = legitimate page auto-forwards to malicious site. Domain Hijacking = attacker takes over the domain registration itself.
T/FQuestion 29 of 30A — True
DNS sinkholes, which are basically the same technique as DNS poisoning, can be used by security professionals to prevent users from carrying out botnet instructions.
Explanation: TRUE. DNS sinkhole uses the same technique as DNS poisoning but defensively — security teams redirect botnet C2 domain queries to a controlled server (sinkhole), preventing compromised machines from receiving instructions. Same technique, opposite intent.
MCQuestion 30 of 30B
In a split-tunnel VPN, traffic is handled as follows:
Explanation: Split-tunnel VPN: org/protected network traffic → VPN tunnel; general internet browsing → direct (not through VPN). Full-tunnel VPN routes ALL traffic through VPN. Split-tunnel is more efficient but offers less visibility into user activity.
