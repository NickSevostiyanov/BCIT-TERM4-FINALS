MCQuestion 1 of 30C
Which Docker command creates AND starts a new container from an image?
Explanation: docker run = CREATE + START a container. docker pull = download image only. docker build = build image from Dockerfile. docker create = create without starting. You chose docker pull on Quiz 1 — that only downloads, doesn't run anything.
MCQuestion 2 of 30B
What is the primary purpose of a Docker volume?
Explanation: Docker volume purpose = persist data even after the container is deleted. Bind mounting is one mechanism for a volume, but not the purpose. On Quiz 1 you chose "bind mount a directory" — that's the mechanism, not the purpose.
MCQuestion 3 of 30C
Which Docker command is used to connect to (or run a command in) a running container?
Explanation: docker exec = run a command/connect to a RUNNING container. On Quiz 1 you chose docker run — that creates a NEW container, it doesn't connect to an existing one. Example: docker exec -it mycontainer /bin/bash
MCQuestion 4 of 30B
What is the difference between a Docker image and a Docker container?
Explanation: Image = blueprint (read-only template). Container = runtime instance (running/stopped instance created from an image). On Quiz 1 you chose A (reversed them). Remember: image is like a class, container is like an object instance.
T/FQuestion 5 of 30B — False
Docker is an orchestration tool used to manage containerized applications at scale.
Explanation: FALSE. Docker is a container platform (build, ship, run containers). Kubernetes is the orchestration tool. On Quiz 1 you chose True — this was wrong. Docker does NOT orchestrate containers at scale; K8s does.
MCQuestion 6 of 30C
Which command removes a Docker image from the local system?
Explanation: docker rmi = remove image (the 'i' = image). docker rm = remove container. You got this wrong on Quiz 1 — chose docker rm. Remember: rmi for image, rm for container.
MCQuestion 7 of 30B
What is the purpose of a Dockerfile?
Explanation: Dockerfile = stores build instructions (FROM, RUN, COPY, EXPOSE, CMD, etc.). You run docker build against a Dockerfile to create an image. It defines how the image is assembled.
MCQuestion 8 of 30B
What is the purpose of a Docker container restart policy?
Explanation: Restart policy controls container behavior on exit. --restart always = restart whenever it exits or when Docker daemon restarts. This ensures high availability for containerized services.
MCQuestion 9 of 30C
How do you expose a port from a Docker container to the host machine?
Explanation: Port mapping (-p flag) exposes container ports to the host. Example: docker run -p 8080:80 nginx maps host port 8080 to container port 80. Volumes are for data, not ports.
MCQuestion 10 of 30B
Which command is used to create a new Docker network?
Explanation: docker network create [name] creates a new Docker network. docker network ls lists networks. Networks allow containers to communicate with each other. You got this right on Quiz 1.
T/FQuestion 11 of 30B — False
docker pull both downloads an image AND starts a new container from it.
Explanation: FALSE. docker pull ONLY downloads the image from a registry — it does NOT create or start a container. Use docker run to create AND start. Note: docker run automatically pulls if the image isn't local.
MCQuestion 12 of 30C
In Kubernetes, what is the smallest deployable unit that can be created and managed?
Explanation: Pod = smallest deployable unit in Kubernetes. A Pod can contain one or more containers sharing network and storage. You got this right on Quiz 2 and Quiz 3.
MCQuestion 13 of 30D
What Kubernetes object declares the desired state for a set of Pods and facilitates rolling updates and rollbacks?
Explanation: Deployment = declares desired state + manages rolling updates and rollbacks. ReplicaSet only ensures replica count. Deployment is the higher-level object you interact with; it creates and manages ReplicaSets automatically.
MCQuestion 14 of 30B
How do Kubernetes Services primarily enable network access to a set of Pods?
Explanation: K8s Service = stable IP + DNS name routing to Pods. Pod IPs change when Pods restart — Services abstract this with a stable endpoint. You got this right on Quiz 2.
MCQuestion 15 of 30C
What is the primary purpose of Kubernetes?
Explanation: Kubernetes = automates deployment, scaling, and management of containerized applications. Docker builds/runs containers; K8s manages them at scale across clusters of machines.
MCQuestion 16 of 30C
Which managed Kubernetes service is offered by Amazon Web Services (AWS)?
Explanation: EKS = Elastic Kubernetes Service = AWS. GKE = Google. AKS = Azure. DOKS = DigitalOcean. OpenShift = Red Hat (not AWS). You got this right on Quiz 2.
MCQuestion 17 of 30D
Which Kubernetes object directly ensures a specified number of identical Pod replicas are running at all times?
Explanation: ReplicaSet = directly ensures specified number of Pod replicas and automatically replaces failed Pods. Deployment manages ReplicaSets at a higher level. The ReplicaSet is the mechanism; the Deployment is the controller.
MCQuestion 18 of 30C
A Kubernetes Service of type NodePort exposes a service on a static port on each node. What is the typical range for these NodePort ports?
Explanation: NodePort range = 30000–32767. This is a fixed Kubernetes specification. Memorize it exactly — it shows up on exams (you got this right on Quiz 3).
MCQuestion 19 of 30B
What is the primary role of the kube-apiserver component in Kubernetes?
Explanation: kube-apiserver = exposes the Kubernetes API (front-end of the control plane). kube-scheduler = assigns Pods to nodes. Network policy enforcement is done by the CNI plugin. This was wrong on Quiz 3 (chose "manage network policies").
MCQuestion 20 of 30C
What is the default Kubernetes Service type that exposes the service on an internal IP, only reachable from within the cluster?
Explanation: ClusterIP = DEFAULT Service type. Only reachable within the cluster (internal access). NodePort = external via node static port. LoadBalancer = external via cloud LB. You got this wrong on Quiz 3 and Quiz 4 — confusing ClusterIP/LoadBalancer/NodePort roles.
MCQuestion 21 of 30D
Which Kubernetes Service type exposes an application externally by provisioning a cloud provider load balancer?
Explanation: LoadBalancer = provisions an external load balancer from the cloud provider (AWS ELB, GCP Cloud LB, etc.). You chose NodePort on Quiz 4 — NodePort only exposes on a static port on each node, not via a cloud LB.
MCQuestion 22 of 30B
When installing Kubernetes on Docker Desktop, which statement best describes it?
Explanation: Docker Desktop K8s = lightweight single-node cluster integrated within the Docker Desktop app. Not production-grade, not multi-master, no cloud account required. Good for learning and local development.
T/FQuestion 23 of 30A — True
Containers within the same Kubernetes Pod share a network namespace and can communicate with each other via localhost.
Explanation: TRUE. Containers in the same Pod share the same network namespace (same IP address). They communicate via localhost on different ports. They also share storage volumes if configured. This is why initContainers can write to shared volumes that the main container reads.
MCQuestion 24 of 30C
What is the main purpose of a PersistentVolumeClaim (PVC) in Kubernetes?
Explanation: PVC = request/claim storage from available PVs. PV = the actual storage resource (provisioned by admin or dynamically). PVC = the application's request. K8s binds a PVC to a matching PV automatically.
T/FQuestion 25 of 30B — False
Data stored on a properly mounted PersistentVolume is deleted when the Pod using it is deleted and recreated.
Explanation: FALSE. Data on a PV persists independently of the Pod lifecycle. Deleting/recreating the Pod does NOT delete PV data. That's the entire purpose of PersistentVolumes — to outlive the Pods that use them. Only deleting the PV itself would remove the data.
MCQuestion 26 of 30C
Which field in a PersistentVolume specification indicates how much storage is available to be claimed?
Explanation: capacity field = storage size available (e.g., capacity: storage: 1Gi). You chose volumeMode on Quiz 4 — volumeMode specifies Filesystem or Block mode, not size. accessModes = how the volume can be accessed (ReadWriteOnce, etc.).
MCQuestion 27 of 30D
Which Kubernetes Deployment strategy is commonly used to update Pods gradually with minimal downtime?
Explanation: RollingUpdate = gradual replacement of old Pods with new ones — minimal downtime (default strategy). Recreate = terminates all old Pods first, then creates new ones (causes downtime). ImmediateReplace is not a real K8s strategy.
MCQuestion 28 of 30B
What is the primary purpose of a Kubernetes ConfigMap?
Explanation: ConfigMap = store non-sensitive config as key-value pairs (e.g., environment variables, config files). For sensitive data (passwords, API keys), use Secrets. Never put passwords in a ConfigMap.
MCQuestion 29 of 30C
An initContainer in a Kubernetes Pod __________.
Explanation: initContainer = runs to completion BEFORE the main container starts. Common use: write config files to a shared volume, wait for a dependency. A = sidecar (runs alongside). You chose "single container + kubectl cp" on Quiz 4 instead of the initContainer pattern.
MCQuestion 30 of 30B
What is the primary difference between a Kubernetes Secret and a ConfigMap?
Explanation: Secret = sensitive data stored base64-encoded (NOT encrypted by default in K8s). ConfigMap = non-sensitive config. They are NOT interchangeable. Note: D is backwards — ConfigMaps are NOT encrypted; Secrets are base64-encoded (not plain text, but also not truly encrypted unless etcd encryption is configured).
