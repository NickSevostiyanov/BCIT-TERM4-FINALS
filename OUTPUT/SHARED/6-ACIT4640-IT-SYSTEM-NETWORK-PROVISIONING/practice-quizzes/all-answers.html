MCQ1 / 25B
1. What is the correct AWS CLI command pattern?
Explanation: AWS CLI always follows: aws [options] <service> <operation> [parameters]. Example: aws ec2 describe-instances — service = ec2, operation = describe-instances.
T/FQ2 / 25True
2. aws configure stores credentials in ~/.aws/credentials and config in ~/.aws/config.
Explanation: TRUE. ~/.aws/credentials holds aws_access_key_id and aws_secret_access_key. ~/.aws/config holds region and output format. Named profiles can be used with --profile dev.
MCQ3 / 25C
3. You want to verify AWS CLI permissions WITHOUT making any changes. Which flag do you use?
Explanation: --dry-run checks if you have the necessary permissions without actually executing the API call. --check is the Ansible equivalent. --no-paginate disables pagination on list commands.
MCQ4 / 25B
4. After creating a .pem key file, what permission must you set before using it with SSH?
Explanation: chmod 600 = owner read+write only. SSH refuses to use key files that are group- or world-readable (e.g., 644). The course teaches 600 as the standard. chmod 400 (read-only) also works in practice.
MCQ5 / 25C
5. What is the default SSH username for an Ubuntu EC2 instance?
Explanation: Ubuntu AMIs → ubuntu. Amazon Linux AMIs → ec2-user. This is a classic exam trap. The Packer source AMI in the course uses Ubuntu with ssh_username = "ubuntu".
MCQ6 / 25B
6. What is the correct SSH command to connect to an Ubuntu EC2 instance using a .pem key?
Explanation: -i = identity file (private key). -k is not a valid SSH flag for key files. The full command: ssh -i demo-key.pem ubuntu@PUBLIC_IP.
MCQ7 / 25C
7. Which AWS resource allows instances in a VPC to communicate with the internet?
Explanation: The Internet Gateway (IGW) is attached to the VPC and enables internet connectivity. You then add a route in the Route Table pointing 0.0.0.0/0 to the IGW. Without an IGW, the subnet is private even if you add routes.
MCQ8 / 25C
8. What CIDR block notation represents ALL IP addresses (anywhere on the internet)?
Explanation: 0.0.0.0/0 = all IPs. Used in security group rules for public access (SSH from anywhere, HTTP from anywhere). /32 = exactly 1 IP address. /16 = 65,536 IPs. /24 = 256 IPs.
T/FQ9 / 25True
9. A security group in AWS is stateful — if inbound traffic is allowed, the return traffic is automatically allowed.
Explanation: TRUE. Security groups are stateful — response traffic is tracked and automatically allowed. Network ACLs (NACLs) are stateless and require explicit inbound AND outbound rules. Know the difference.
MCQ10 / 25C
10. What is the correct Terraform workflow order?
Explanation: init → downloads providers/modules (required first). plan → preview changes (dry run). apply → create/update resources. destroy → tear down. You cannot run plan or apply without init first.
MCQ11 / 25C
11. Which Terraform command shows what changes WILL be made WITHOUT actually making them?
Explanation: terraform plan is the dry-run for Terraform — it shows + (create), ~ (modify), - (destroy) but makes NO changes. terraform apply --dry-run does not exist. terraform validate only checks HCL syntax.
T/FQ12 / 25True
12. Terraform is a declarative IaC tool — you describe the desired state and Terraform figures out how to achieve it.
Explanation: TRUE. Terraform = declarative (describe WHAT you want). AWS CLI = imperative (specify HOW each step). Declarative tools calculate the diff between current state and desired state automatically.
MCQ13 / 25B
13. Where does Terraform track the state of managed infrastructure?
Explanation: terraform.tfstate is the state file — it maps your HCL resources to real AWS resource IDs. NEVER delete it. If lost, Terraform cannot track, update, or destroy existing resources.
MCQ14 / 25C
14. In a Terraform project, which file typically contains resource definitions and provider configuration?
Explanation: Convention: main.tf = provider + resource blocks. variables.tf = variable declarations. outputs.tf = output values. providers.tf = optional separate provider config. terraform.tfstate = auto-generated state (not edited manually).
MCQ15 / 25B
15. Ansible is an agentless tool. How does it communicate with managed hosts?
Explanation: Ansible is agentless — no software needs to be installed on managed nodes. It connects over SSH using credentials defined in the inventory file (ansible_user, ansible_ssh_private_key_file). Push-based architecture.
MCQ16 / 25B
16. In an Ansible inventory file, what does the [web] line define?
Explanation: [web] is a group header in the INI-format inventory. All hosts listed below it belong to the "web" group. Target with: ansible web -m ping or hosts: web in a playbook.
MCQ17 / 25B
17. Which Ansible command tests connectivity to all hosts and returns "pong" on success?
Explanation: ansible all -m ping uses the built-in ping module. Success response = pong. Use web instead of all to target a specific group. The -m flag specifies the module.
T/FQ18 / 25True
18. In an Ansible playbook, become: yes means the task will run with elevated (sudo) privileges.
Explanation: TRUE. become: yes enables privilege escalation (equivalent to sudo). Required for tasks like installing packages with apt, managing services, or writing files to system directories like /var/www/html/.
MCQ19 / 25C
19. Which Ansible module is used to install packages on Ubuntu/Debian systems?
Explanation: apt module = Ubuntu/Debian. yum module = RHEL/CentOS/Amazon Linux. The course focuses on Ubuntu EC2 instances. Key parameter: state: present to install, state: absent to remove.
MCQ20 / 25C
20. What does Packer produce as its output?
Explanation: Packer builds machine images (AMIs on AWS). The process: launch a temporary EC2 → run provisioners (shell scripts / Ansible) → create an AMI → terminate the instance. No running instances remain after the build.
MCQ21 / 25C
21. What is the purpose of packer validate .?
Explanation: Packer commands: packer init . = download plugins. packer validate . = check HCL syntax (no build). packer build . = execute the full build and produce the AMI.
MCQ22 / 25B
22. What is Infrastructure as Code (IaC)?
Explanation: IaC = infrastructure defined in code files (version-controlled, repeatable, automatable). Benefits: consistency, peer review, rollback, automation. Declarative IaC (Terraform) describes desired state. Imperative (CLI scripts) describes steps.
MCQ23 / 25C
23. A VPC has CIDR block 10.0.0.0/16. A subnet has CIDR 10.0.1.0/24. How many IP addresses does the subnet provide?
Explanation: /24 = 2^(32-24) = 2^8 = 256 addresses. /16 = 65,536. /32 = 1. Formula: 2^(32 - prefix). Quick reference: /24 = 256, /16 = 65,536, /32 = 1 (single host), 0.0.0.0/0 = all IPs.
MCQ24 / 25D
24. Which tool is best described as a configuration management tool (installs/configures software on existing servers)?
Explanation: Tool roles: Ansible = configuration management (configures running servers). Terraform = infrastructure provisioning (creates VMs, networks). Packer = image building (bakes AMIs). AWS CLI = imperative cloud management (any AWS API call).
T/FQ25 / 25True
25. terraform destroy tears down all infrastructure managed by the current Terraform state.
Explanation: TRUE. terraform destroy reads terraform.tfstate and destroys every resource it tracks. It prompts for confirmation (or use -auto-approve to skip). This is why the state file must be preserved — without it, Terraform can't destroy cleanly.
