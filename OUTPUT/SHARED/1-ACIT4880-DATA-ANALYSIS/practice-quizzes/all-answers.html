Part 1 — NumPy & Pandas (Q1–Q6)
Question 1True/False
df.head() without arguments returns the first 10 rows by default.
○ True
✓ False
Answer: FALSE. df.head() returns the first 5 rows by default. Use df.head(10) to get 10. This specific fact appeared on Quiz 1 as a wrong answer trap.
Question 2True/False
df.shape returns a tuple of (rows, columns).
✓ True
○ False
Answer: TRUE. df.shape returns a tuple (n_rows, n_columns). For example, (500, 12) means 500 rows and 12 features.
Question 3Multiple Choice
What happens when you run:
arr1 = np.array([1, 2, 3]) arr2 = arr1 arr2[0] = 99
○ Error: cannot modify arr2
✓ Both arr1 and arr2 become [99, 2, 3]
○ arr1 becomes [99, 2, 3] and arr2 remains [1, 2, 3]
○ arr1 remains [1, 2, 3] and arr2 becomes [99, 2, 3]
Answer: Both arrays become [99, 2, 3]. arr2 = arr1 creates a reference — both names point to the same memory. Changing one changes both. Use arr2 = arr1.copy() for a true independent copy. This exact question appeared on Quiz 1.
Question 4Multiple Choice
What will df.iloc[2:5, 1:3] return?
✓ Rows 2 to 4 and columns 1 to 2
○ Rows 2 to 4 and columns 1 to 3
○ Rows 2 to 5 and columns 1 to 2
○ Rows 2 to 5 (inclusive) and columns 1 to 3 (inclusive)
Answer: Rows 2–4, columns 1–2. Python slices are exclusive of the upper bound. 2:5 → indices 2, 3, 4. 1:3 → indices 1, 2. This exact question appeared on Quiz 1 — the most common mistake is thinking the upper bound is inclusive.
Question 5Multiple Choice
What is the primary difference between a NumPy array and a Pandas Series?
✓ Series has an index and can hold different data types; arrays are homogeneous
○ Arrays are always faster
○ Series can only hold numeric data
○ No significant difference
Answer: Series has index + mixed types; arrays are homogeneous. NumPy arrays must be a single dtype. Pandas Series adds a labeled index and can hold object dtype (mixed). Series can also hold strings, booleans, and mixed types as object dtype.
Question 6True/False
The iloc[] method in pandas is used for label-based indexing.
○ True
✓ False
Answer: FALSE. iloc[] = integer location (position). loc[] = label based. Example: df.iloc[0] gets the first row by position. df.loc['rowlabel'] gets by label. This was tested directly on Quiz 1.
Part 2 — Data Preprocessing & EDA (Q7–Q12)
Question 7Multiple Choice
Which transformation is most appropriate when dealing with highly skewed data?
○ Standardization (Z-score normalization)
✓ Logarithmic transformation
○ One-hot encoding
○ Min-max scaling
Answer: Logarithmic transformation. Log compresses the tail of a skewed distribution, making it more symmetric/normal. Z-score and min-max change scale but not shape. One-hot is for categoricals. This appeared on Quiz 2.
Question 8True/False
Outliers should always be removed from a dataset to ensure better model performance.
○ True
✓ False
Answer: FALSE. Outliers are NOT always errors. A billionaire in a salary dataset is an outlier but valid. Blind removal loses information. Investigate: is it data entry error? Legitimate extreme? Only then decide to remove, cap, or keep. This appeared on Quiz 2.
Question 9Multiple Choice
In supervised learning, the target variable is also known as:
✓ The dependent variable
○ The predictor variable
○ The independent variable
○ The explanatory variable
Answer: Dependent variable. The target is what we predict — it "depends" on the input features. Input features are the independent/predictor/explanatory variables. This appeared on Quiz 2. Mixing these up is a common mistake.
Question 10Multiple Choice
In the context of data preprocessing, what does GIGO stand for?
○ General Information, General Output
✓ Garbage In, Garbage Out
○ Great Input, Great Output
○ Grouped Input, Grouped Output
Answer: Garbage In, Garbage Out. The fundamental principle of data quality: even the most sophisticated algorithm fails if fed dirty, incorrect, or irrelevant data. This was directly tested on Quiz 2.
Question 11True/False
Substituting missing data with the mean is always good because measures of spread are not affected.
○ True
✓ False
Answer: FALSE. Mean imputation DOES distort measures of spread. By replacing missing values with the mean, you artificially cluster values around the center, reducing variance and distorting the true distribution. This appeared on Quiz 2.
Question 12Multiple Choice
Which of the following is NOT a typical activity in EDA?
○ Developing an initial idea of possible associations among predictors
○ Examining important interrelationships between attributes
○ Identifying interesting subsets of the observations
✓ Testing a specific statistical hypothesis
Answer: Testing a specific hypothesis. EDA is open-ended exploration. Formal hypothesis testing belongs to inferential statistics. EDA uncovers structure; inferential stats confirms or rejects specific claims with statistical rigor. This appeared on Quiz 2.
Part 3 — Machine Learning Algorithms (Q13–Q23)
Question 13Multiple Choice
In kNN, what happens when you choose a very low value of k?
○ The model underfits the training data
✓ The model overfits — too sensitive to noise
○ The model ignores all training data
○ No effect — k doesn't matter
Answer: Overfitting. k=1 → classify by the single nearest neighbor → memorizes training noise → overfits. Large k → smoother decision boundary → underfitting. Optimal k is found via cross-validation. Rule of thumb: try k = √n.
Question 14Multiple Choice
What technique does Random Forest use to create diversity among its trees?
○ Each tree uses the full dataset and all features
○ Boosting — trees built sequentially to fix errors
✓ Bagging — random subsets of data AND features for each tree
○ Pruning — each tree simplified after training
Answer: Bagging. Bootstrap Aggregating = train each tree on a random sample (with replacement) of training data AND a random subset of features. This creates decorrelated trees. Final answer = majority vote (classification) or mean (regression). Boosting is used in AdaBoost/GBM/XGBoost — fundamentally different.
Question 15Multiple Choice
Which metric is most appropriate for an imbalanced classification dataset?
○ Accuracy
✓ F1 Score
○ Mean Squared Error
○ R² Score
Answer: F1 Score. With 95% class 0 and 5% class 1, a model predicting all class 0 = 95% accuracy. F1 = 0 because it caught no positives. F1 balances precision (quality of positive predictions) and recall (coverage of actual positives). Use F1, AUC-ROC, or per-class precision/recall for imbalanced datasets.
Question 16True/False
SMOTE generates synthetic samples for the minority class to handle imbalanced datasets.
✓ True
○ False
Answer: TRUE. SMOTE (Synthetic Minority Oversampling Technique) creates new minority examples by interpolating between existing ones. Better than simple duplication because it adds diversity. Alternative: undersampling (remove majority class), or class_weight='balanced' in sklearn models.
Question 17Multiple Choice
In a Decision Tree, what does "information gain" measure?
○ The total number of splits in the tree
✓ The reduction in entropy after a split on a feature
○ The increase in the number of features used
○ The depth of the resulting tree
Answer: Reduction in entropy. IG = Entropy(parent) − Σ weighted Entropy(children). High IG = that feature creates purer subgroups. The tree algorithm greedily picks the feature with highest IG at each node. Gini impurity is an alternative to entropy for the same purpose.
Question 18Multiple Choice
In K-Means clustering, the "Elbow Method" is used to:
✓ Determine the optimal number of clusters (k)
○ Find the centroid with the lowest variance
○ Remove outliers before clustering
○ Determine the distance metric to use
Answer: Find optimal k. Plot inertia (WCSS — within-cluster sum of squares) vs k. At first, adding more clusters greatly reduces inertia. Eventually, the improvement flattens. The "elbow" = point of diminishing returns = optimal k. Example: sharp drop from k=1→3, then flattens at k=4 → use k=4.
Question 19Multiple Choice
Which regression evaluation metric(s) are in the same units as the target variable?
○ MSE (Mean Squared Error) only
○ R² Score only
○ RMSE only
✓ Both MAE and RMSE are in the same units as the target
Answer: Both MAE and RMSE. MAE = mean of |y - ŷ|. RMSE = √MSE. Both are in original units (e.g., dollars if predicting salary). MSE is in squared units (dollars²). R² is unitless (0–1). When explaining to stakeholders, prefer MAE or RMSE over MSE.
Question 20Multiple Choice
In an SVM, what are "support vectors"?
○ All data points used during training
✓ The data points closest to the decision hyperplane
○ Vectors used to transform features into higher dimensions
○ Features with the highest variance
Answer: Closest data points to the hyperplane. Support vectors define the margin. The SVM objective is to maximize the margin between the two classes. Only support vectors matter — removing any other training point would not change the model. The kernel trick maps data to higher dimensions to find a separating hyperplane.
Question 21True/False
kNN is a parametric learning algorithm — it builds an explicit model during training.
○ True
✓ False
Answer: FALSE. kNN is non-parametric and a lazy learner. During "training" it just stores the dataset — no model is built. At prediction time, it computes distances to all training points and votes. This makes training instant but prediction slow for large datasets.
Question 22Multiple Choice
Recall (Sensitivity) is calculated as:
○ TP / (TP + FP) — this is Precision
✓ TP / (TP + FN)
○ (TP + TN) / Total — this is Accuracy
○ 2 × Precision × Recall / (P + R) — this is F1
Answer: TP / (TP + FN). Recall = of all actual positives, how many did we correctly identify? FN = false negatives = cases we missed. High recall is critical in medical diagnosis (don't miss cancer patients). High precision = when we predict positive, we're usually right.
Question 23Multiple Choice
Which statement about Hierarchical Clustering is correct?
○ You must specify k before running it
○ It always uses Euclidean distance only
✓ It builds a dendrogram that can be cut at any level to get clusters
○ It cannot handle more than 100 data points
Answer: Dendrogram cut at any level. Key advantage over K-Means: no k required upfront. View the dendrogram and decide. Linkage methods: single (closest), complete (farthest), average, Ward (minimizes variance — most common). Agglomerative (bottom-up) is more common than divisive (top-down).
Part 4 — Model Tuning & Evaluation (Q24–Q30)
Question 24True/False
Feature scaling is required before applying kNN.
✓ True
○ False
Answer: TRUE. kNN computes Euclidean distance. A feature like "salary" (50,000) would completely dominate "age" (25–65) without scaling. Always normalize or standardize features before kNN, SVM, or any distance-based algorithm. Tree-based models (DT, RF) do NOT require scaling.
Question 25Multiple Choice
Which scikit-learn method performs exhaustive search over ALL combinations of hyperparameters?
○ RandomizedSearchCV
○ CrossValidateCV (not a real class)
✓ GridSearchCV
○ BayesianSearchCV
Answer: GridSearchCV. Grid search tries every possible combination from the parameter grid. Slow but thorough. RandomizedSearchCV samples a random subset — faster. Both use cross-validation internally. For production hyperparameter tuning, Bayesian optimization (e.g., Optuna) is smarter but not standard sklearn.
Question 26Multiple Choice
What does R² = 0 indicate about a regression model?
○ The model perfectly predicts the target
✓ The model performs no better than predicting the mean of the target
○ The model has a 0% error rate
○ The model has 0% accuracy
Answer: No better than mean baseline. R² measures proportion of variance explained. R²=1 = perfect. R²=0 = model explains nothing beyond the mean. R² can be NEGATIVE if the model is worse than the mean baseline. It is NOT a percentage — it is a ratio of explained variance.
Question 27Multiple Choice
Which describes HIGH VARIANCE (overfitting) in a model?
○ Poor performance on both training and test sets
✓ Excellent training performance but poor test performance
○ Equal performance on training and test regardless of complexity
○ The model predicts the same value for all inputs
Answer: Great train, poor test. High bias (underfitting) = poor on BOTH. High variance (overfitting) = great on train (memorized it), poor on test (can't generalize). Solutions for overfitting: regularization (L1/L2), cross-validation, more training data, simpler model, ensemble (Random Forest).
Question 28True/False
Decision trees are prone to overfitting and can be controlled with max_depth or pruning.
✓ True
○ False
Answer: TRUE. An unrestricted decision tree grows until it perfectly fits training data (pure leaves = overfit). Controls: max_depth, min_samples_split, min_samples_leaf, and pruning. Random Forest solves this by averaging many trees on different data subsets.
Question 29Multiple Choice
Which imputation technique is most susceptible to introducing bias?
○ Predictive modeling-based imputation
○ k-Nearest Neighbors imputation
✓ Mean imputation
○ Multiple imputation
Answer: Mean imputation. Replacing ALL missing values with a single constant (the mean) creates artificial clustering at the mean, reducing variance and distorting correlations. Multiple imputation is the gold standard. KNN uses relationships between features to impute more intelligently. This was directly tested on Quiz 2.
Question 30Multiple Choice
Which transformations can ALL stabilize variance and make data more normally distributed?
○ Logarithmic transformation only
○ Square root transformation only
○ Inverse transformation only
✓ All of the above (log, square root, and inverse)
Answer: All of the above. All three are power transformations that compress large values and expand small values, reducing right skew. Log is most aggressive (best for heavy skew). Square root is gentler. Inverse is for very extreme skew. This exact "all of the above" pattern appeared on Quiz 2 (Q17) — this course frequently uses it as the correct answer.
